# 设计规则集

> 本文件是 idea-to-github-pages Skill 的自包含设计规则。编写时参考了以下开源项目的思路，所有规则已转化为可直接映射到 HTML/CSS 的操作级指引。

---

## 致谢与出处

本规则集参考了以下开源项目的核心方法论，特此致谢：

- **[design-taste-frontend](https://github.com/codebuddy-plugins)** — 三档位设计系统（VARIANCE / MOTION_INTENSITY / VISUAL_DENSITY）、配色校准规则、字体禁用清单、布局多样性规则。作者通过系统的反模式清单和上下文感知的设计引擎，为前端设计提供了可操作的品质控制框架。
- **[taste-design](https://labs.google.com/stitch)** — Google Stitch 语义化设计系统（DESIGN.md 格式）。将设计拆解为氛围、色彩角色、组件行为、动效哲学、反模式的自然语言描述，为「AI 读懂设计」提供了范本。
- **[frontend-design](https://github.com/codebuddy-plugins)** — 大胆美学方向指引。强调差异化、字体性格、空间组合的艺术性，反对「安全但平庸」的 AI 默认审美。
- **[ui-ux-pro-max](https://github.com/codebuddy-plugins)** — 67 种风格 × 96 套配色 × 57 组字体配对的搜索式设计数据库。提供了「按产品类型匹配风格」的系统化方法，以及严格的交付前检查清单。

以下规则融合了上述项目的核心思想，并简化为适合「一句话描述 → 配色+字体+布局+动效」的直接指导。

## 一、风格→配色映射

### 暖色系

用户说「温暖的 / 日系的 / 咖啡馆感觉的 / 手作」

```
主色：#E07A3D（焦橙）或 #C45A3C（砖红）
辅色：#F5E6D3（奶油白）或 #FAF6F0（暖纸）
背景：#FFFAF5（暖白）或 #FEF9F3
文字：#2D2420（深棕）
强调文字：#B85C38
```

**禁止：** 不要用纯白色底（太冷），不要用亮蓝色（破坏暖感）。

### 冷色系 / 科技感

用户说「科技公司风格的 / 专业冷静的 / 咨询法律」

```
主色：#1E3A5F（深蓝）或 #2563EB（电蓝）
辅色：#E8F4FD（淡蓝灰）
背景：#FFFFFF 或 #F8FAFC
文字：#0F172A
强调文字：#1D4ED8
```

### 暗色背景

用户说「深色底的 / 程序员风格的 / 游戏 / 酷炫」

```
主色：#4ADE80（亮绿）或 #38BDF8（亮蓝）
辅色：#1E293B（深石板灰）
背景：#0F172A（深蓝灰）
文字：#E2E8F0
强调文字：#60A5FA
```

**必须：** 深色背景上的白色文字不能太亮 — 用带灰度的白（#E2E8F0 而非 #FFFFFF），避免刺眼。

### 亮色清爽

用户说「干干净净的 / 杂志风的 / 现代品牌」

```
主色：#059669（翡翠绿）或 #0284C7（天蓝）
辅色：#F0FDF4（淡绿）或 #F0F9FF（淡蓝）
背景：#FFFFFF
文字：#18181B
强调文字：#0369A1
```

### 活泼创意

用户说「有活力的 / 吸眼球的 / 年轻的 / 好玩的」

```
主色：#EC4899（玫红）或 #F59E0B（明黄）
辅色：高饱和对比色
背景：#FFFFFF 或 #FFF7ED（活力暖）
文字：#18181B
强调文字：#DB2777
```

**警告：** 高饱和活泼风格容易翻车。控制强调色只在按钮和小面积使用。

## 二、字体选择

### 默认（推荐，适合绝大多数页面）

```
标题：Geist / Outfit / Satoshi（选一个）
正文：系统字体栈（-apple-system, PingFang SC, Noto Sans SC, sans-serif）
等宽（代码/数字）：JetBrains Mono / Fira Code
```

### 中文适配

中文内容必须包含中文字体栈：

```
--sans: "PingFang SC", "Noto Sans SC", "Microsoft YaHei", -apple-system, sans-serif;
```

Geist/Outfit 等西文字体只当装饰标题用，正文一定是中文字体栈。

### 禁用字体

- **不要用 Inter**（太像 AI 生成的默认）
- 不要用 Roboto/Arial/微软雅黑作为**唯一**选择（太普通）
- 除非用户明确要衬线风格，不要用衬线字体（宋体等）做默认

### 字号规范

```
Hero 标题：clamp(2.5rem, 5vw, 4rem)  — 最大两行
正文：1rem（16px），行高 1.7，每行最多 65 个中文字
小标题：clamp(1.25rem, 3vw, 2rem)
辅助文字：0.875rem
```

## 三、布局规则

### Hero（第一屏）

- 标题最多 2 行，超出就缩短文字
- 副标题最多 20 个字
- 最多 1 个按钮（加最多 1 个辅助链接）
- 页面打开时必须完整看到，不能需要滚动

### Section 排列

- **禁止** 连续 3 个「左图右文→右图左文→左图右文」交替排列。最多 2 节就换。
- 每个 Section 用不同的布局形式：
  - 居中单栏（叙事）
  - 左右分屏（对比）
  - 2-3 列卡片（并列信息）
  - 全宽背景（视觉冲击）
- 8 个 Section 的页面，至少用 4 种不同布局

### 间距

```
Section 垂直间距：clamp(3rem, 8vw, 6rem)
Section 水平内边距：clamp(1rem, 5vw, 3rem)
卡片间距：1-1.5rem
```

### 圆角统一

一个页面只选一种圆角风格：
- 全直角：`border-radius: 0`（硬朗专业）
- 全圆润：`border-radius: 12px`（柔和亲和）
- 全胶囊：按钮等用 `border-radius: 999px`

**禁止** 混用 — 有的圆有的方，看起来像没做完。

---

## 四、响应式（手机适配）

### 断点

```
手机 < 768px
平板 768-1024px
桌面 > 1024px
```

### 手机规则

- 所有多列布局在手机上变成单列
- 字号不能小于 14px
- 按钮最少 44×44px（手指能点到）
- Hero 用 `min-height: 100dvh`，不要用 `100vh`（iOS Safari 地址栏会导致跳动）
- 手机上**绝对不能**左右滚动

---

## 五、动效

### 级别

根据用户选择的风格映射：

```
专业/可信/冷色 → 极简动效：只在 hover 时轻微变色
暖色/清爽/活泼 → 适度动效：页面加载时淡入，滚动时浮现
花哨/创意/游戏 → 不做（GitHub Pages 不适合复杂动效）
```

### 规则

- 只用 CSS `transition` 和 `opacity`（不用 GSAP/Motion 等框架）
- 动效时长 200-400ms
- 必须加 `prefers-reduced-motion` 支持（无障碍）
- **禁止**：无限循环动画、弹跳、抖动、旋转文字

---

## 六、色彩纪律

- 一个页面最多 1 个强调色
- 饱和度不超过 80%（太艳了刺眼）
- 不在同一个页面混用暖灰和冷灰
- 不在正文用纯黑（#000000），用深灰（#18181B）
- 卡片阴影不要纯黑，要保持颜色与底色协调

---

## 七、禁止事项

| 类别 | 禁止 |
|------|------|
| 配色 | AI 紫/蓝发光渐变、纯黑 #000000、高饱和度大色块 |
| 字体 | Inter 作为默认、混用衬线无衬线 |
| 布局 | 三等分相同卡片 × N、居中 Hero（非编辑/宣言类页面）、手机横向滚动 |
| 文字 | Emoji 当图标、空泛套话（「赋能」「非凡」「极致」）、假数据、假统计数据 |
| 图片 | 破碎的 Unsplash 链接、用 div 模拟图片 |
| 动效 | 无限循环弹跳/闪烁、不兼容 prefers-reduced-motion |

---
