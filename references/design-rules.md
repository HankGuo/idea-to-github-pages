# 前端设计规则（design-taste-frontend 精简版）

> 从 design-taste-frontend Skill 提取的核心设计规则。当该 Skill 不可用时作为 fallback。

---

## 0. 读取 Brief

在写任何代码前，推断用户真正想要的设计方向：

1. **页面类型** — 产品介绍页 / 个人主页 / 活动页 / 作品集 / 其他
2. **风格词** — "简约" / "科技感" / "温暖" / "专业" / "有冲击力" / 等
3. **受众** — B2B 客户 / 消费者 / 招聘者 / 社群成员
4. **已有品牌资产** — logo、品牌色、字体
5. **约束** — 无障碍优先 / 公开部门 / 受监管行业

### Design Read

在生成代码前，输出一句话定位：

> "定位：<页面类型>，面向<受众>，<风格>语言"

示例：「定位：AI SaaS 产品页，面向技术采购者，Linear 式极简冷静风格」

---

## 1. 三个档位（核心配置）

| 档位 | 范围 | 说明 |
|------|------|------|
| DESIGN_VARIANCE | 1-10 | 1=完美对称，10=艺术感混乱 |
| MOTION_INTENSITY | 1-10 | 1=静态，10=电影级物理动效 |
| VISUAL_DENSITY | 1-10 | 1=画廊级留白，2=信息密集 |

**默认基线：8 / 6 / 4**

按风格推断：

| 风格信号 | VARIANCE | MOTION | DENSITY |
|---------|----------|--------|---------|
| 极简/编辑/Linear | 5-6 | 3-4 | 2-3 |
| 高端消费品/Apple | 7-8 | 5-7 | 3-4 |
| 花哨/实验/Awwwards | 9-10 | 8-10 | 3-4 |
| Landing/作品集(默认) | 7-9 | 6-8 | 3-5 |
| 信任优先/公共部门 | 3-4 | 2-3 | 4-5 |

## 2. 字体规则

### 无衬线（默认）
- **默认选型：** 避免 Inter 作为默认。首选 Geist、Outfit、Cabinet Grotesk、Satoshi
- **Inter 可用场景：** 用户明确要求中性/标准/Linear 风格，或公共部门/无障碍优先站点
- **中文匹配：** Noto Sans SC、PingFang SC、Microsoft YaHei（系统字体栈）

### 衬线体（非常不建议作为默认）
- 仅在以下情况使用：品牌明确指定衬线字体、真正编辑/奢侈品/手稿风格
- **禁止默认用：** Fraunces、Instrument Serif

### 排版规范
- 标题：`text-4xl md:text-6xl tracking-tighter leading-none`（CSS等效：`font-size: clamp(2.25rem, 4vw, 3.75rem); letter-spacing: -0.03em; line-height: 1`）
- 正文：`text-base leading-relaxed max-w-[65ch]`（CSS等效：`font-size: 1rem; line-height: 1.75; max-width: 65ch`）
- 标题斜体降部必须留空间：含 `y g j p q` 的斜体词 → `line-height: 1.1` 最小

---

## 3. 配色规则

### 基础规则
- **最多 1 个强调色。** 饱和度 < 80%
- **AI 紫/蓝发光禁止作为默认。** 使用中性底 + 高对比单一强调色（翡翠绿、电蓝、深玫红、焦橙等）
- **一个项目一个调色板。** 不在同一项目中混用暖灰和冷灰
- **色彩一致性锁定：** 选定强调色后全页面统一。不存在「暖灰页面第 7 节突然变蓝色 CTA」

### 阴影
- 绝不用纯黑阴影。根据背景色调色
- DENSITY > 7 时禁止卡片容器

### 高端消费品调色板禁令
禁止默认使用以下配色作为高端消费品页面的底/强调色：
- 背景：`#f5f1ea` `#f7f5f1` `#fbf8f1` `#efeae0`（暖纸/奶油/粉笔色系）
- 强调色：`#b08947` `#b6553a` `#9a2436`（黄铜/黏土/牛血色系）
- 替代方案：冷奢(银灰+铬)、深林(深绿+骨白)、黑棕、钴蓝+奶油、赤陶+石板灰

## 4. 布局规则

### 对齐
- VARIANCE > 4 时禁止居中 Hero。使用分屏(50/50)、左对齐内容+右对齐资源、不对称留白
- 编辑/宣言/发布公告类页面允许居中 Hero

### Hero
- Hero 必须适配首屏。标题桌面端最多 2 行，副标题最多 20 词、最多 4 行
- Hero 顶部内边距最大 `6rem` (≈96px)
- Hero 文字元素最多 4 个：眉题(可选) + 标题 + 副标题 + CTA(1主要+最多1次要)
- **Hero 堆叠纪律：** 禁止在 Hero 区域堆信任徽标行、功能列表、品牌条、价格预告

### 布局多样性
- 同一布局族最多出现 1 次。3 栏卡片图 / 全宽引言 / 分屏图文 → 各只能出现一次
- Zigzag 交替最大 2 节。第 3 节必须换布局
- 眉题(UPPERCASE 小标签)最大出现频率：每 3 个 section 最多 1 个
- 分屏标题禁止作为默认：「左大标题+右小说明」模式优先堆叠标题

### 形状一致性
- 全页面统一圆角尺度：全直角(0)、全柔和(12-16px)、全胶囊(全圆角)。
- 混合体系需要明确规则

---

## 5. 响应式

- 断点：`sm 640px` `md 768px` `lg 1024px` `xl 1280px` `2xl 1536px`
- 页面容器：`max-width: 1400px; margin: 0 auto` 或 `max-width: 80rem`
- **禁止 `height: 100vh`：** 移动端 Safari 地址栏会跳动，使用 `min-height: 100dvh`
- **Grid 优先：** 不用 `calc()` 计算 flex 百分比，用 grid 列

---

## 6. 反默认纪律

- 不使用 AI 紫渐变、居中 hero + 深色网格、三等分功能卡片、全站玻璃态
- 不使用无限循环微观动画
- 不使用 Inter + slate-900 配色
- 不使用 emoji 作为 UI 元素
- 不手写复杂 SVG 插图——用图标库或简单几何图形

---

## 7. 图标

允许：Phosphor Icons、Tabler Icons、Radix Icons（通过 CDN 或内联 SVG）
不建议：Lucide
禁止：手写图标路径
一个项目一个图标族，不混用

HTML 中使用方式：从对应 CDN 引用或直接內联 SVG path
